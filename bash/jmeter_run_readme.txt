
1. С помощью переменных задать в сценарии такие параметры:
 - jmeter_env_archname, jmeter_env_archpath - источник архива с тестовой инфраструктурой на файловой системе.
 - FUEL_IP - адрес fuel-master.
 - login/password - параметры входа на fuel-master
 - SNAPSHOT - номер тестируемой сборки (важно при отправке отчёта в TestRail)

2. Если не планируется выполнять тестирование различных специфичных apache-конфигураций для Keystone, то
цикл "for config_item in $KEYSTONE_CONFIGS; do" необходимо полностью закомментировать, а переменную KEYSTONE_CONFIGS
установить значнием "6,1" (что соответствует стандартному, начиная с MU 9.1, числу процессов на котроллер и числу потоков на
процесс в конфигурации apache для keystone).
Установка данной переменной, кроме прочего, необходима также для фиксации информации о тестируемой конфигурации в TestRail.

ВАЖНО!
------
Скрипт расчитан на работу в условиях, когда каждому compute-серверу кластера назначен публчный IP.

   * tests_extra_long.tar.gz - тесты по 59 мин. 30 сек.
 - детальное описание сценариев: https://docs.google.com/document/d/1jTyCXcr91X02NOGp4NKkezJ2LE1FEfDH4jZmJQEIj_E/edit#heading=h.mx987y5hsw0h

В случае, если необходимо изменить длительность выполнения теста на какую-то другую, следует в JMX-файле сценария найти строку
параметра "<stringProp name="Hold">", задать (в секундах) необходимую длительность выполнения теста и поместить изменения в
новую версию архива с инфраструктурой.
